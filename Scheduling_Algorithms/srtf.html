Shortest Remaining Time First (SRTF) CPU Scheduling Simulation

Explanation

Shortest Remaining Time First (SRTF) is a preemptive version of the Shortest Job First (SJF) algorithm. In SRTF, the process with the smallest remaining execution time is selected for execution. The running process can be preempted if a new process arrives with a smaller burst time.

Key Characteristics:

  • Preemptive: The current process can be interrupted if a new process with shorter remaining time arrives.
  • Optimal average waiting time: Theoretically provides the minimum average waiting time among all scheduling algorithms.
  • Priority: Based on the shortest remaining burst time.

Algorithm Steps:

  1. When a new process arrives, compare its burst time with the remaining burst time of the running process.
  2. If the new process has a shorter burst time, preempt the current process and start executing the new one.
  3. Otherwise, continue with the current process.
  4. Repeat this check at every process arrival and completion.

Performance Metrics:

  • Waiting Time (WT): Time a process waits in the ready queue before getting CPU.
  • Turnaround Time (TAT): Total time taken from submission to completion.
  • Response Time: Time from submission until the first CPU response.

Turnaround Time = Completion Time - Arrival Time

Waiting Time = Turnaround Time - Burst Time

Calculation:

  • Average Waiting Time = (10 + 0 + 14 + 2) / 4 = 26 / 4 = 6.5 units
  • Average Turnaround Time = (16 + 4 + 22 + 5) / 4 = 47 / 4 = 11.75 units

Sequence of Events:

  1. P1 arrives at time 0 and starts execution.
  2. P2 arrives at time 1 with burst time 4, which is less than P1's remaining time (5), so P1 is preempted and P2 starts.
  3. P3 arrives at time 2, but its burst time (8) is greater than P2's remaining time (3), so P2 continues.
  4. P4 arrives at time 3, but its burst time (3) is equal to P2's remaining time (2), so P2 continues.
  5. P2 completes at time 5, then P4 starts as it has the shortest remaining time (3).
  6. P4 completes at time 8, then P1 resumes (remaining time: 5).
  7. P1 completes at time 16, then P3 starts and completes at time 24.

Advantages:

  • Provides minimum average waiting time among all scheduling algorithms.
  • Responsive to short processes.
  • Good for interactive systems where quick response time is important.

Disadvantages:

  • Starvation: Long processes may never get executed if shorter processes keep arriving.
  • Overhead: Requires constant monitoring of remaining burst times and frequent context switches.
  • Unpredictable: Cannot accurately predict completion times due to preemptions.
  • Requires burst time knowledge: Implementation requires knowing burst times in advance, which is often not possible in real systems.

Best Used For:

  • Interactive systems that require quick response times.
  • Systems where process burst times are known in advance.
  • Environments where minimizing average waiting time is a priority.
