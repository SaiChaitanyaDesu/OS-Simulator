Earliest Deadline First (EDF) CPU Scheduling Simulation

Explanation

Earliest Deadline First (EDF) is a dynamic priority-driven scheduling algorithm used in real-time systems. As the name suggests, processes with earlier deadlines are given higher priority.

Key Characteristics:

  • Preemptive: If a process with an earlier deadline arrives, the currently running process is preempted.
  • Dynamic priority: Process priorities change based on their deadlines.
  • Optimal: EDF is optimal for scheduling periodic tasks on a single processor.

Algorithm Steps:

  1. Assign priorities to processes based on their absolute deadlines (earlier deadline = higher priority).
  2. At any scheduling point, execute the process with the highest priority (earliest deadline).
  3. If a new process arrives with an earlier deadline, preempt the currently running process.
  4. Continue until all processes complete.

Performance Metrics:

  • Waiting Time (WT): Time a process waits in the ready queue before getting CPU.
  • Turnaround Time (TAT): Total time taken from submission to completion.
  • Response Time: Time from arrival until first execution.
  • Deadline Miss Ratio: Percentage of processes that miss their deadlines.

Sequence of Events:

  1. P1 arrives at time 0 and gets the CPU. Executes for 3 time units.
  2. P2 arrives at time 2 with deadline 10, but P1 has earlier deadline (7).
  3. P3 arrives at time 4 with deadline 8, after P1 completes. Since P3's deadline (8) is earlier than P2's (10), P3 gets the CPU.
  4. P4 arrives at time 6 with deadline 15. Since its deadline is later than both P2 and P3, it waits.
  5. P3 completes at time 5+2=7, then P2 gets the CPU and runs to completion.
  6. P4 runs last and completes at time 14.

Advantages:

  • Optimal for scheduling periodic tasks on a single processor.
  • Dynamic priority assignment based on deadlines.
  • High processor utilization.
  • Minimizes deadline misses.

Disadvantages:

  • Overhead due to frequent preemptions.
  • No consideration of task importance or criticality.
  • Performance degrades under overload conditions.
  • Requires accurate deadline information for all tasks.

Best Used For:

  • Real-time systems where meeting deadlines is critical.
  • Systems where task priorities can change dynamically.
  • Systems where processor utilization needs to be maximized.
